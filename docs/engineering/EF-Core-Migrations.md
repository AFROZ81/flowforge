# Entity Framework Core Migrations

This document describes how Entity Framework Core (EF Core) Migrations are used in FlowForge.

EF Core Migrations are used to create, update, and maintain the SQL Server database schema throughout the development of the project.

# Overview

FlowForge uses Entity Framework Core Code First with SQL Server.

The database schema is managed using EF Core Migrations.

Whenever the domain model changes, a new migration is created to keep the database synchronized with the application's models.

---

# Why Migrations?

Migrations provide a version-controlled history of database changes.

Benefits include:

- Track database schema changes
- Create databases automatically
- Update existing databases safely
- Maintain consistent environments
- Work well with source control

---

# Current Database Provider

FlowForge currently uses:

```text
SQL Server
```

The DbContext is:

```text
ApplicationDbContext
```

Entity Framework Core is responsible for generating the database schema based on the configured entities.

---

# Creating a Migration

Whenever entity configurations or models change, create a new migration.

Example:

```bash
dotnet ef migrations add InitialCreate
```

Additional examples:

```bash
dotnet ef migrations add AddProjects

dotnet ef migrations add AddBoards

dotnet ef migrations add UpdateProjectConfiguration
```

Migration names should clearly describe the purpose of the change.

---

# Applying a Migration

After creating a migration, apply it to the database.

```bash
dotnet ef database update
```

This updates the SQL Server database to the latest migration.

---

# Updating Existing Models

When modifying an entity:

1. Update the entity.
2. Update any related Entity Framework configuration if required.
3. Create a new migration.
4. Apply the migration.

Example workflow:

```text
Modify Entity

↓

Create Migration

↓

Review Migration

↓

Update Database
```

---

# Removing a Migration

If a migration has not yet been applied to the database, it can be removed.

```bash
dotnet ef migrations remove
```

This removes the most recent migration.

---

# Migration Files

Migration files are automatically generated by Entity Framework Core.

Typical contents include:

- Table creation
- Column updates
- Index creation
- Foreign keys
- Constraints

These files should be committed to source control.

---

# Database Synchronization

All developers working on the project should use the latest migrations.

Before starting development:

```bash
git pull

dotnet ef database update
```

This ensures the local database matches the current project.

---

# Best Practices

✔ Create a migration for every schema change.

✔ Use meaningful migration names.

✔ Review generated migrations before applying them.

✔ Commit migration files to source control.

✔ Apply migrations after pulling the latest code.

✔ Keep the database synchronized with the current migration history.

---

# Common Commands

Create Migration

```bash
dotnet ef migrations add MigrationName
```

Update Database

```bash
dotnet ef database update
```

Remove Last Migration

```bash
dotnet ef migrations remove
```

List Migrations

```bash
dotnet ef migrations list
```

Generate SQL Script

```bash
dotnet ef migrations script
```

# Summary

FlowForge uses Entity Framework Core Migrations to manage the SQL Server database schema.

Every schema change is tracked through migrations, providing a reliable and version-controlled approach to database evolution throughout the project lifecycle.
